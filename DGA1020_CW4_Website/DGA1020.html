Introduction


For this task we were required to 3D sculpt in Blender a "3D Still Life Fruit Bowl", details on the sculpts were a requirement. Then we were tasked with animating the bowl rotating, and creating an MP4 animation of the 3D still life.

Last but not least we needed to create a 2D character sprite animation in Piskel, specifying a theme and animating the following actions: i. Idle (2 frames) ii. Walk (~6 frames) iii. Run (~6 frames) iv. Jump (~3 frames) v. Die (~4 frames) vi. Fall (~2 frames) . Then setting-up the sprite sheet and animation state-manager in Unity, to finally export a screen capture MP4 including the Unity Package.




Development



Blender

First before creating our video, the subject had to be modelled in blender. The Bowl was not a hard task, first a circle is added then using upward extrusions a pillar is created. Outward extrusions allow us to create a lamp shape, then sections of the bowl are selected and brought out to form a proper bowl shape. It is then later smoothened out, and given it's abstract shape using proportional editing.

The fruits were a much harder task for me, I had to do some research on youtube in order to get an idea on how to create these shapes.

Then it was time to animate the fruit bowl, a transparent globe is added around the fruit bowl. Then two lights are also added, in render cycles is used for realistic shading. Then using object relationships, the globe is made to rotate with the camera, eventually upon playing the keyframes are created and we got our final output.




Unity

Creating the sprite sheet required use of Piskel and Unity animating software. Using the sprite editor in Unity, the sprite sheet was sliced into individual squares using the correct column and rows. Each of the squares are named in order such as Corsair_Jump_01, to be used in the animator. As you can see the sprite editor has created individual pieces from the sprite sheet, a game object is created and the sprite renderer is loaded. The individual pieces are also then connected together into animations such as Corsair_Idle, Corsair_Run and Corsair_Die. To begin animating first the animator component is added in our game object, the animator controller is then loaded into the assets. Later the actual animator(state machine) is opened, this controls the behaviour of an object in a game. The animations are added into the state machine, each of the animations are connected together in a loop sequence. The animations are then added and played in the animation preview , the samples are changed to lower ones for a smoother look. Later the animation is arranged as required in the preview, to be recorded and processed for the final output.
